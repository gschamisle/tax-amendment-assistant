# 법인세법·소득세법 병행개정 매뉴얼

이 문서는 법인세법과 소득세법 사이에서 동일하거나 유사한 정책 목적을 가진 조문을 개정할 때, 병행개정 필요성을 우선 검토해야 하는 조문을 정리한다.

현재 앱에서는 아래 확정 매핑을 GPT 추론보다 먼저 적용한다. 즉, 기준 조문이 확인되면 모델 판단과 무관하게 병행개정 후보로 표시한다.

## 3. 감가상각비 하위법령 연쇄 검토

감가상각비는 법률 조문 자체는 확정 매핑하지만, 시행령은 감가상각자산의 범위, 상각방법, 내용연수, 즉시상각, 상각부인액 처리, 명세서 제출 등 여러 조문으로 나뉜다. 따라서 법인세법 제23조 또는 소득세법 제33조제1항제6호를 개정할 때에는 다음 조문군을 함께 검토한다.

| 구분 | 법인세법 시행령 | 소득세법 시행령 | 검토 취지 |
| --- | --- | --- | --- |
| 감가상각자산·상각액 기본 구조 | 제24조, 제25조, 제26조 | 제62조 | 감가상각자산의 범위, 손비 또는 필요경비 계상, 상각범위액 계산 구조 |
| 내용연수·상각률 | 제28조, 제29조 | 제63조 | 내용연수, 상각률, 내용연수 특례 또는 변경 |
| 상각방법 | 제27조 | 제64조, 제65조, 제66조 | 상각방법 신고, 변경, 정액법·정률법 등 계산 방법 |
| 특수 계산 | 제26조의2, 제26조의3, 제29조의2 | 제67조, 제68조, 제70조, 제71조, 제73조 | 종전·기준 감가상각비, 중고자산, 즉시상각, 의제상각, 잔존가액, 상각시부인 |
| 상각부인액·명세서 | 제32조, 제33조, 제34조 | 제73조, 제73조의2 | 상각부인액 처리, 감가상각비 명세서 제출 및 세칙 |

## 4. 코드 기반 유사도 매칭 규칙

확정 매핑에 없는 조문이라도 법인세법과 소득세법 사이에서는 다음 표현을 같은 개념으로 정규화한 뒤, 조문 제목과 핵심 표현이 유사하면 GPT 호출 전에 코드로 병행 후보를 반환한다.

| 법인세법 표현 | 소득세법 대응 표현 |
| --- | --- |
| 손금 | 필요경비 |
| 손금불산입 | 필요경비불산입 |
| 손금산입 | 필요경비산입 |
| 손금에 산입하지 아니 | 필요경비에 산입하지 아니 |
| 손금에 산입 | 필요경비에 산입 |
| 각 사업연도, 사업연도 | 과세기간 |
| 내국법인, 법인 | 거주자 |
| 업무와 관련 없는 비용 | 업무무관비용 |

유사도 매칭에서 특히 강하게 보는 핵심어는 다음과 같다.

- 업무용승용차
- 필요경비불산입 또는 손금불산입
- 필요경비산입 또는 손금산입
- 업무무관비용
- 기부금
- 기업업무추진비 또는 접대비
- 결손금 또는 이월결손금
- 재해손실세액공제
- 감가상각비
- 한도
- 특례
- 세액공제, 세액감면
- 익금, 총수입금액

## 5. 실무 적용 원칙

1. 법인세법 또는 소득세법의 위 확정 매핑 조문을 개정하면, 대응 조문은 반드시 병행개정 후보에 포함한다.
2. 금액, 비율, 한도, 계산식, 적용시기, 이월 또는 공제기간이 바뀌면 하위법령도 함께 검토한다.
3. 법인세법에서 "손금"으로 표현된 항목은 소득세법의 "필요경비" 대응 규정이 있는지 확인한다.
4. 법인세법에서 "사업연도" 기준을 바꾸면 소득세법의 "과세기간" 기준도 확인한다.
5. 법률 조문만 개정하더라도 시행령·시행규칙에 위임된 금액이나 계산식이 있으면 연쇄개정 필요성이 높다.

## 6. 법인세법·부가가치세법 연쇄 검토

법인세법과 부가가치세법은 동일 정책의 “병행 개정”이라기보다, 사업자등록·증빙·감가상각자산·매입세액 등에서 **의존·연쇄** 관계가 많다. 아래는 코드(`_PARALLEL_ARTICLE_HINTS`)로 우선 검토 후보에 올리는 대표 연결이다.

| 기준(법인세법) | 병행 검토(부가가치세법) | 검토 취지 |
| --- | --- | --- |
| 제11조 | 제8조 | 납세지 변경신고와 사업자등록·변경신고 |
| 제111조 | 제8조 | 사업자등록 |
| 제23조 | 제41조, 제43조, 제44조 | 감가상각자산·매입세액 재계산·공제특례 |
| 제75조의5, 제116조 | 제32조, 제35조 | 세금계산서·지출증명 |
| 제75조의8, 제120조의3 | 제54조 | 세금계산서합계표 제출 |
| 법인세법 시행령 제22조 | 제39조, 제42조(법), 시행령 제74조·제75조 | 매입세액 손금산입과 부가세 공제 구조 |

부가가치세법을 개정할 때는 위 표의 **역방향** 힌트도 `_PARALLEL_ARTICLE_HINTS`에 반영되어 있다.

## 7. 아직 수동 검토가 필요한 영역

아래 영역은 코드 유사도 매칭으로 후보를 찾을 수 있지만, 확정 매핑으로 보기에는 조문 구조와 적용 대상이 넓어 수동 검토가 필요하다.

- 익금과 총수입금액 대응 규정
- 자산 평가, 준비금, 충당금 관련 규정
- 특수관계인, 부당행위계산, 이전가격 관련 규정
- 비과세·감면·세액공제 중 조세특례제한법과 함께 얽히는 규정
- 신고, 납부, 가산세, 원천징수처럼 절차 규정이 섞이는 조문

## 8. 조세특례제한법 연동

조특법 조문이 다른 세법을 **명시 인용**한 연결은 `data/special-tax-parallel-candidates.json`에 저장한다. 앱은 `core/special_tax_hints.py`가 이 파일을 읽어 병행 검사 시 GPT보다 먼저 적용한다.

| 데이터 | 앱 반영 |
| --- | --- |
| `source: citation`, `confidence: high` | 병행 후보에 반영 (`[명시 인용]` 표기) |
| `keyword`, `gpt`, 조문 미특정 | 검토용만 (자동 반영 안 함) |

- **조특법 개정 시**: JSON 정방향 → 인용된 소득세법·법인세법·부가가치세법 조문
- **세법 개정 시**: JSON 역방향 → 해당 조문을 인용하는 조특법 특례 조문

갱신: `uv run python scripts/build_special_tax_links.py` 실행 후 앱 재시작.

## 9. 코드 반영 위치

- 확정 병행 매핑: `core/cross_ref_checker.py`의 `_PARALLEL_ARTICLE_HINTS`
- 조특법 명시 인용: `core/special_tax_hints.py` + `data/special-tax-parallel-candidates.json`
- 법인세법·소득세법 표현 정규화: `core/cross_ref_checker.py`의 `_TAX_PAIR_REPLACEMENTS`
- 유사도 핵심어: `core/cross_ref_checker.py`의 `_TAX_PAIR_TOPIC_TERMS`
- 병행 법령 후보군: `config.py`의 `PARALLEL_LAWS`

확정 매핑을 추가하려면 `_PARALLEL_ARTICLE_HINTS`에 다음 형태로 항목을 넣는다.

```python
("법인세법", "소득세법", "조번호"): [
    {
        "article": "제OO조",
        "reason": "병행 검토가 필요한 이유",
    }
]
```

조번호는 법제처 API 형식에 맞춰 `27의2`처럼 저장하고, 표시용 조문은 `제27조의2`처럼 작성한다.

## 10. 연간·시행일 갱신

1. `uv run python scripts/check_law_freshness.py` — `data/law-snapshot-manifest.json` 대비 변경 여부
2. 변경 시 `uv run python scripts/check_law_freshness.py --update-manifest`
3. `uv run python scripts/build_special_tax_links.py` — 조특법 명시 인용 JSON 재생성
4. 확정 매핑(`_PARALLEL_ARTICLE_HINTS`)·이 문서 수동 검토
5. Streamlit 재시작(법령 텍스트 캐시 갱신)

앱 기동 시 manifest와 현행본이 다르면 상단에 경고가 표시된다.
